# Анализатор страниц (Page Analyzer)

### Hexlet tests and linter status:
[![Actions Status](https://github.com/KrasnopevtsevAlexey/php-project-9/actions/workflows/hexlet-check.yml/badge.svg)](https://github.com/KrasnopevtsevAlexey/php-project-9/actions)

«Анализатор страниц» — это веб-приложение (SEO-инструмент) для анализа указанных сайтов на соответствие базовым требованиям поисковой оптимизации. Программа валидирует и сохраняет URL, проверяет доступность страниц, а также парсит и выводит ключевые теги: `h1`, `title` и мета-тег `description`.

---

## 🚀 Системные требования

Для запуска и тестирования приложения вам понадобятся:
* **Операционная система:** Linux, macOS, или Windows (с установленным WSL2)
* **Docker:** Версия 20.10.0 или выше
* **Docker Compose:** Версия 2.0.0 или выше

*(Установка PHP, Composer или СУБД на локальный компьютер не требуется — всё окружение полностью изолировано и разворачивается внутри контейнеров).*

---

## 🛠️ Инструкция по установке и запуску

Выполните последовательно следующие команды в терминале вашей системы:

### 1. Клонирование репозитория
```bash
git clone https://github.com
cd php-project-9
```

### 2. Подготовка конфигурации
Создайте файл настроек окружения `.env` из шаблона (при необходимости скорректируйте порты):
```bash
cp .env.example .env
```

### 3. Сборка и запуск через Docker Compose
Запустите сборку проекта в фоновом режиме. Docker автоматически скачает необходимые образы, установит зависимости Composer/NPM, выполнит миграции базы данных SQLite и запустит веб-сервер Caddy:
```bash
docker-compose up -d --build
```

После успешного завершения команды приложение станет доступно в браузере по адресу: **`http://localhost:8080`** (или по порту, заданному в вашем `.env`).

---

## 👨‍💻 Использование приложения

1. Откройте главную страницу в браузере.
2. Введите корректный URL-адрес сайта (например, `https://hexlet.io`) в форму и нажмите кнопку **«Проверить»**.
3. Если сайт добавляется впервые, он запишется в базу данных. Вы будете перенаправлены на индивидуальную страницу этого URL.
4. Нажмите кнопку **«Запустить проверку»** — приложение совершит HTTP-запрос к сайту с помощью Guzzle, извлечет SEO-метрики через DomCrawler и обновит историю проверок.

---

## ⚙️ Полезные команды для разработки

* **Остановить контейнеры:** `docker-compose down`
* **Просмотр логов в реальном времени:** `docker-compose logs -f`
* **Запуск тестов (Playwright):** `make test`
* **Проверка кода линтером (PSR-12):** `composer exec phpcs`
